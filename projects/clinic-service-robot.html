Clinic Service Robot (Internship)

During my internship at Smart Healthcare Technology, Inc., our intern team co-developed an AI-powered clinic customer service system for Cheng Hsin General Hospital (CHGH, 振興醫院). The project aimed to improve clinic efficiency and patient experience through a practical conversational service workflow.

The system was built with Botpress and integrated with the company's next-generation clinic cloud APIs. The implementation included OAuth-based authentication, API integration for core clinic services, and JavaScript-based data processing modules. It supports key services including appointment booking, cancellation, queue and visit-progress checking, registration status checking, symptom inquiry, doctor schedule lookup, and FAQ support.

What We Built

Natural-Language Registration Flow

A version that allows users to describe booking needs in natural language, with language model support for intent understanding.

Step-by-Step Registration Flow

A guided version that collects required booking fields through conversation, stores variables, and assembles JSON payloads for API submission.

Core Service Functions

Appointment actions, doctor scheduling, symptom lookup, queue updates, and general clinic information support were combined in one assistant entry point.

User Experience Goal

The project focused on making medical service interaction easier and more accessible, improving convenience and overall patient satisfaction.

Tech Stack

Botpress, JavaScript, OAuth-based authentication, clinic cloud API integration, JSON-based request handling, and deployment on the company's 163 server environment.
